# GeoNeo AI Docs

This folder defines what GeoNeo AI is building, how the product should behave, and how engineering should implement and maintain it.

## Document Index

- `CEO_TECHNICAL_BRIEF.md`: CEO-facing technical overview with an AI-ready context pack.
- `SITE_GOALS.md`: Business and product outcomes the site is designed to achieve.
- `MVP.md`: Current minimum viable product scope, constraints, and release criteria.
- `PRODUCT_RULES.md`: Tier behavior, lead capture rules, and internal/admin expectations.
- `ARCHITECTURE.md`: High-level system design, endpoints, and data flow.
- `PROGRAMMING_GUIDE.md`: Practical implementation guide for backend/frontend/testing.
- `ENGINEERING_RULES.md`: Non-negotiable engineering standards and guardrails.

## Primary Audience

- Product owner
- Engineers
- QA
- Internal operations/admin users

## Update Rule

When product behavior changes (form fields, package tiers, API response shape, lead storage), update these docs in the same change set.
